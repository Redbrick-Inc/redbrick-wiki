# 디버깅 및 오류 해결 방법

에러를 찾는 여러 가지 방법이 있습니다.

<Steps>

### 콘솔

무언가 작동하지 않는다면, `console.log()`를 사용하여 해당 부분의 코드가 작동하는지 확인해볼 수 있습니다.  

<Callout type="info">
    콘솔창 여는 법
    <br/>
    <center>![debug_console-window](/images/snippet/tips/debug/debug_console-window.png) F12 또는 브라우저 화면에서 우클릭후 검사 선택 > 우측에 열린 콘솔 창에서 상단 Console 탭 선택</center>
</Callout>

##### 주의사항
1. 에러 발생 이후에 콘솔창을 확인하면 에러의 내용이 표시되지 않을 수 있습니다. 실시간으로 에러 내용을 확인하고 싶다면 콘솔창을 열어놓은 채로 게임을 테스트 해주세요.  
2. 콘솔창에 나오는 에러 중에는 레드브릭 엔진과 관련이 없거나 플레이에 지장을 주지 않는 에러들도 있습니다. 콘솔창에 에러가 나왔다고 해서 무조건 문제가 되는 것은 아닙니다.    

<br/>
<Tabs items={["예시 1", "예시 2"]} defaultIndex="0">

    <Tabs.Tab>

        ```js showLineNumbers copy {6}
        let sum = 0;

        for (let i = 1; i <= 25; i++) {
            sum += i;
            console.log(`sum = ${sum}, i = ${i}`);
        }
        ```

### 에러 팝업

<Callout type="error">
  에러 팝업  
  <br/>
  <center>![debug_error-popup](/images/snippet/tips/debug/debug_error-popup.png) 에러 발생 시 나오는 에러 팝업</center>  
</Callout>

##### 주의사항
에러 팝업은 에러가 발생했다고 해서 항상 나타나지는 않습니다.  
만약 에러 팝업이 나오지 않는다면 콘솔창을 확인해 주세요.

### 프로젝트 저장 및 다시 열기

모든 것이 정상이라고 생각되고 오류를 찾을 수 없다면, "파일로 저장" 옵션을 사용하여 프로젝트를 `.json`파일로 저장합니다.

그 후 새 프로젝트를 생성하고 "파일에서 열기"를 사용하여 저장한 파일을 엽니다.

마지막으로 프로젝트를 다시 실행하여 문제가 해결되었는지 확인하십시오.

</Callout>
</Steps>
