## 🧑‍💻 여러명이 작업하는 법

<Callout type="warning">
  레드브릭 엔진은 실시간 공동 작업을 지원하지 않습니다.  
  따라서 팀 프로젝트를 진행할 때 한 사람의 계정으로 여러 명이 로그인하여 작업을 할 수는 있지만, 팀원들 각자의 수정 내용들이 실시간 연동, 동기화되지는 않습니다.  
  따라서 추천하는 방법은 크게 세 가지가 있습니다.  
</Callout>

### 방법 1 : 베이스 프로젝트를 파일로 저장한 후, 팀원들 각자 따로따로 작업한 뒤에 하나의 프로젝트로 합치기  

1. 팀원들의 작업물들을 취합할 대표자 한 명을 정합니다.  
2. 대표자는 처음에 베이스가 될 프로젝트를 생성하고, 코딩을 제외한 맵 구성(GUI, 오브젝트 배치 등)을 미리 완료해 놓습니다.  
3. 대표자는 베이스 프로젝트를 파일로 저장하고, 팀원들에게 나누어줍니다.  
4. 팀원들은 각자 자신의 계정에서 파일을 열고, 자신이 맡은 파트에 대한 작업을 진행합니다.  
5. 작업이 끝난 팀원은 자신이 작업한 프로젝트를 파일로 저장해 대표자에게 전달합니다.  
6. 대표자는 팀원들의 작업물을 전달받은 후, 자신의 프로젝트에 해당 작업들을 똑같이 따라 해서 적용합니다.  

### 방법 2 : 여러명이 각자 따로 프로젝트를 진행 한 뒤, 포탈을 통해 연결  

1. 프로젝트를 기획할 때, 게임을 파트별로 나눠 각자 개발할 파트를 정합니다.  
2. 팀원들은 각자 프로젝트를 생성해 자신이 맡은 파트를 작업합니다.  
3. 작업이 완료된 프로젝트를 [포탈](../snippet/redbrick-extension/portal)을 통해 서로 연결합니다.  


### 방법 3 : 팀원들이 모두 한 사람의 계정으로 로그인한 뒤, 순차적으로 작업하기  

1. 팀원들의 작업 순서를 정한 뒤, 모두 한 사람의 계정으로 로그인합니다.  
2. 첫 번째 작업자부터 작업을 진행합니다.  
3. 첫 번째 작업자의 작업이 끝났으면 프로젝트를 저장합니다.  
4. 두 번째 작업자가 프로젝트를 불러온 후, 이어서 작업합니다.  
5. 두 번째 작업자의 작업이 끝났으면 프로젝트를 저장합니다.  
6. 위의 절차를 반복하며 순서대로 작업합니다.  




## 💾 웹 기반인 레드브릭 엔진은 중간중간 저장이 필수!

### Auto Save 기능을 사용한다면

- **Auto Save를 활성화하면 5분마다 자동으로 프로젝트가 저장됩니다.**

<Callout type="warning">
  **주의사항**: 여러 명이 한명의 계정으로 로그인하여 작업할 때, Auto Save를
  켜놓으면 해당 작업자의 내용으로 프로젝트가 저장될 수 있습니다.
</Callout>

### **보다 안전하게 저장하기 위해서는**

- **Auto Save를 비활성화하시고 자주 수동으로 Save해주세요.**

<Callout type="info">
  **중요한 작업이 끝났을 경우:** 더 안전한 작업을 위해서 **Save to File**로
  기기에 파일로 저장해두시는 것을 추천드립니다.
